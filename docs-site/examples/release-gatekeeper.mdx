## Overview

The Release Risk Gatekeeper is the flagship reagent-flow demo. Three cooperating sub-agents evaluate whether a software release is safe to deploy:

```
        get_release_info                assess_risk              make_decision
User -> Gatherer agent --[handoff]--> Assessor agent --[handoff]--> Decider agent
            |                            |                             |
        session A                    session B                     session C
        (parent)                    (child of A)                 (child of B)
```

Each phase is its own reagent-flow session, linked via `parent_trace_id` and `handoff_context`. Contract assertions validate the data flowing between agents.

## The three scenarios

### Scenario 1 — Green path

The full pipeline runs for release `v2.3.1` (risky: open P0, recent rollback). Every contract passes:

```python
# Tool output contract on the gatherer
gatherer.assert_tool_output_matches("get_release_info", schema=RELEASE_INFO_SCHEMA)

# Handoff contract: gatherer -> assessor
assessor.assert_handoff_received(gatherer)
assessor.assert_handoff_matches(schema=RELEASE_INFO_SCHEMA)
assessor.assert_called_times("assess_risk", min=1, max=1)

# Handoff contract: assessor -> decider
decider.assert_handoff_received(assessor)
decider.assert_handoff_matches(schema=RISK_ASSESSMENT_SCHEMA)
decider.assert_tool_output_matches("make_decision", schema=DECISION_SCHEMA)
```

The decider blocks the release. Each session's trace is recorded as a golden baseline.

### Scenario 2 — Broken handoff caught

The gatherer is swapped for a **drifted variant** that renames `issues.open_p0` to `issues.p0_count` and drops `issues.open_p1` — simulating an upstream refactor gone wrong.

The downstream assessor still runs — the LLM is lenient about missing fields — but `assert_handoff_matches` rejects the drifted payload:

```
handoff field 'issues.open_p0': missing from data
```

In CI this fails the test. The PR can't merge until the contract or the gatherer is fixed. **This is the core "caught before release" story.**

### Scenario 3 — Regression diff

The pipeline runs against release `v2.4.0` (clean). The assessor session is diffed against the golden baseline from scenario 1, showing what changed between runs.

## Schemas

```python
RELEASE_INFO_SCHEMA = {
    "release_version": str,
    "branch": str,
    "ci": {
        "pipeline": str,
        "coverage": float,
        "unit_failed": int,
        "integration_failed": int,
        "e2e_failed": int,
    },
    "issues": {
        "open_p0": int,
        "open_p1": int,
        "resolved_this_release": int,
    },
    "deploy_history": [
        {"version": str, "date": str, "status": str, "rollback": bool},
    ],
}

RISK_ASSESSMENT_SCHEMA = {
    "release_version": str,
    "risk_level": str,
    "justification": str,
}

DECISION_SCHEMA = {
    "release_version": str,
    "decision": str,
    "reason": str,
}
```

## What reagent-flow catches

| Check | API | What it catches |
|-------|-----|-----------------|
| Tool output shape | `assert_tool_output_matches` | Upstream API/fixture drift before it reaches another agent |
| Handoff schema | `assert_handoff_matches` | Renamed, missing, or wrongly-typed fields at agent boundaries |
| Parent linkage | `assert_handoff_received` | Child session isn't linked to its expected parent |
| Value preservation | `assert_context_preserved` | A key field (version, user ID) got lost between hops |
| Call counts | `assert_called_times` | Sub-agent skipped its one job or called it more than once |

## Running the demo

### Live (with Gemini API key)

```bash
cd examples/langgraph_demo
export GOOGLE_API_KEY="your-key-here"
uv run python demo.py
```

### As tests (no API key needed)

```bash
cd examples/langgraph_demo
uv run pytest test_demo.py -v
```

The test file exercises the same contract assertions but builds sessions manually with `log_llm_call` / `log_tool_result` — no live LLM calls required.

## Files

| File | Purpose |
|------|---------|
| `tools.py` | The three typed tools, plus a deliberately drifted `get_release_info_drifted` |
| `agent.py` | Sub-agent builders using LangGraph `create_react_agent` |
| `orchestrator.py` | `run_pipeline` — runs the three sessions in sequence and wires handoff contexts |
| `demo.py` | Three scripted scenarios with live LLM calls |
| `test_demo.py` | Same scenarios as pytest tests (no LLM, runs in CI) |
